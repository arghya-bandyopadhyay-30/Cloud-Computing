Amazon EC2 Deep Dive

Meaning of the Name

  • Elastic: Flexible, you can scale up/down based on need.
  • Compute: Processing power (virtual servers) that runs your applications.

Analogy

Renting a dedicated generator for electricity → you don’t own the hardware, but you pay only for what you use.

What is EC2?

  • A web service that provides resizable virtual servers in the cloud.
  • Highly customizable: choose OS, CPU, memory, storage, networking.
  • Pay-as-you-go model: pay only for usage.
  • Scalable: can scale from 1 VM to thousands in minutes.
  • Multi-tenancy allows multiple virtual machines to share resources on the same physical host, with isolation between them.

Instance Types (How much CPU/RAM?)

General Purpose

Analogy: A sedan car → good for most people.

Balanced mix of compute, memory, and networking. Best for web servers, dev/test environments.

Examples: t3, t4g, m5, m6g

Memory Optimized

Analogy: A bus → carries lots of passengers (data).

High memory-to-CPU ratio. Best for in-memory databases, big data analytics.

Examples: r5, r6g, x1, z1d

Storage Optimized

Analogy: A truck → built to carry huge cargo (data).

High, fast local storage with high IOPS. Best for data warehousing, large databases, real-time analytics.

Examples: i3, i3en, d2, h1

Compute Optimized

Analogy: A sports car → built for raw speed.

High-performance processors. Best for gaming servers, HPC, batch processing.

Examples: c5, c6g, c7g

Accelerated Computing

Analogy: A luxury car → specialized for high-end tasks.

Uses GPUs/FPGAs. Best for AI/ML training, 3D rendering, video encoding.

Examples: p3, p4, g4dn, f1

Interacting with AWS

AWS Management Console (Web UI)

A web-based graphical interface to manage AWS services. It lets you search, configure, and monitor resources, and supports multiple logins.

Good for: Beginners, users who prefer a point-and-click interface, and occasional tasks.

Analogy: Like using a bank’s website/app to check your balance and transfer money.

Console Steps to Launch an EC2 Instance

0

Choose Region (Where?)

Determines where your server physically runs. Example: Mumbai (ap-south-1).

1

Choose AMI (What OS/App?)

An Amazon Machine Image is a template with an OS and apps. Options include Linux, Windows, or Marketplace AMIs.

3

Configure Instance

Set network (VPC & subnet), IAM role, monitoring, and launch multiple instances at once.

4

Add Storage

Attach durable EBS volumes or temporary, high-performance instance store volumes.

5

Add Tags

Add key-value metadata (e.g., Name=WebServer1) for billing, grouping, and automation.

6

Configure Security Group (Firewall)

Define inbound/outbound rules to act as a virtual firewall (e.g., allow SSH port 22).

✓

Review & Launch

Check your configuration and launch the instance. Monitor its status on the EC2 Dashboard.

AWS CLI (Command Line Interface)

A tool to manage AWS resources via command line. Supports scripting and automation for repeatable tasks.

aws ec2 run-instances --image-id ami-12345678 --count 1 --instance-type t2.micro

Good for: Power users, developers, and automation via scripts (e.g., in CI/CD pipelines).

Analogy: Like using ATM shortcuts instead of standing in line at the bank counter. Faster if you know the commands.

AWS SDK (Software Development Kit)

Language-specific libraries (APIs) to integrate AWS directly into your applications.

import boto3
s3 = boto3.client('s3')
s3.create_bucket(Bucket='my-test-bucket')

Good for: Developers embedding AWS into apps, building cloud-native apps, and automating resource creation at the code level.

Analogy: Like giving your bank account API access so your finance app can automatically pay bills.
